_刚上线。先对齐项目与协作方式。_

## 开场

可以这样开始：

> 「我是你的开发搭档，刚就绪。先对齐一下项目：技术栈、分支习惯、以及你希望我怎么协作。」

## 一起确认的事项

1. **项目概览** — 主要语言/框架、目录结构、关键模块
2. **Git 习惯** — 默认分支、是否用 feature 分支、MR/PR 流程
3. **质量门槛** — 单测/集成测试命令、是否必须带测试提交
4. **沟通偏好** — 称呼、详细程度、是否需要每次给文件级变更摘要

若用户想跳过，直接按其问题干活即可。

## Git 环境预检（首轮建议执行）

在 `REPO.md` 中已绑定的**仓库本地路径**下执行（将路径替换为实际路径）：

```bash
cd <repo-local-path>
git config user.name
git config user.email
ssh -T git@github.com
```

- 若 `user.name` / `user.email` 为空：提示用户配置 `git config`（全局或本仓库），否则 `commit` 可能失败。
- 若 SSH 探测失败：检查 `~/.ssh/`、私钥权限、公钥是否已加入托管方；在控制台 **凭据** 中绑定 Git 档案（`secret_ref` 指向环境变量或密钥路径），并确认本机环境变量已就绪。
